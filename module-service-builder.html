Problem

AI는 똑똑한데, 결과가 매번 달라집니다.

필요한 건 ‘잘 쓴 한 문장’이 아니라, 같은 품질을 반복해서 뽑아내는 작업 흐름입니다.

  • 매번 새로 짠다

    물어볼 때마다 프롬프트를 처음부터 다시 짜야 합니다.

  • 품질이 들쭉날쭉

    같은 질문도 답의 깊이와 형식이 매번 달라집니다.

  • 우선순위가 없다

    “그래서 뭘 먼저 고치라는 거지?”가 정리되지 않습니다.

  • 개발로 못 넘긴다

    진단을 실제 개발 작업으로 넘기기가 막막합니다.

Solution

분석 → 판단 → 출력 → 검수, 한 흐름으로.

서비스 진단을 네 단계의 고정된 작업 흐름으로 묶었습니다. 누가 돌려도 같은 구조의 결과가 나옵니다.

  1. 01

    분석

    서비스의 목적과 사용자를 먼저 분석합니다.

  2. 02

    판단

    핵심 문제 3~5개를 근거와 우선순위로 정렬합니다.

  3. 03

    출력

    디자인·문구·기능·모바일 UX 개선안을 고정 형식으로 정리합니다.

  4. 04

    검수

    12항목 루브릭으로 결과의 품질을 채점합니다.

FAQ

자주 묻는 질문

그냥 프롬프트 모음 아닌가요?

아닙니다. 분석·판단·출력·검수가 하나로 묶인 작업 시스템이고, 출력 형식이 고정돼 누가 써도 일정한 품질이 나오며, 결과를 점수로 채점하는 루브릭까지 포함됩니다.

어떤 AI에서 쓰나요?

ChatGPT·Claude·Gemini·Cursor·모바일 ChatGPT 각각의 버전이 들어 있습니다.

AI를 잘 몰라도 되나요?

네. 입력 템플릿의 빈칸만 채워서 보내면 됩니다.

결과가 항상 정확한가요?

1차 판단입니다. 모델·버전에 따라 달라질 수 있어, 사람 검토를 전제로 설계했습니다.

전문가가 필요 없어지나요?

아닙니다. 전문가를 고용하기 전, 저비용으로 1차 결과물을 만드는 도구입니다.

환불되나요?

디지털 콘텐츠 특성상, 판매 채널 정책과 관련 법령을 따릅니다.

Sample output

샘플 진단 결과

예시 입력: “동네 베이커리 픽업 예약 웹앱(모바일 위주, 1인 운영).” 아래는 모듈이 실제로 내놓는 출력 형식입니다.

1 · 분석

목적은 ‘예약 픽업 전환’, 주 사용자는 모바일로 빠르게 주문하는 단골. 핵심 동선은 메뉴 → 시간 선택 → 결제 확인입니다.

2 · 판단 (우선순위 문제)

  1. [높음] 픽업 시간 선택이 3단계라 이탈이 큼 — 한 화면 슬롯 선택으로 단축.
  2. [높음] 가격·재고가 결제 직전에야 노출 — 목록 단계에서 미리 표시.
  3. [중간] CTA 문구가 ‘확인’으로 모호 — ‘이 시간에 픽업 예약’으로 구체화.

3 · 출력 (개선안 · 고정 형식)

문제별로 「현상 → 개선안 → 기대효과 → 작업 난이도」를 한 줄로 정리해, 바로 작업 목록으로 옮길 수 있게 합니다.

4 · 검수 (12항목 루브릭)

명확성·우선순위·실행 가능성 등 12항목을 5점 척도로 채점. 예시 결과: 평균 4.1 / 5, 보완 필요 항목 2개를 함께 표시합니다.

실제 결과는 입력 내용과 AI 모델·버전에 따라 달라집니다. 1차 판단이며 사람 검토를 전제로 합니다.
